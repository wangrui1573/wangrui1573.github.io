Safari浏览器 禁用端口说明

发表于2022-05-01|更新于2024-06-26|csdn

苹果系统浏览器在访问一些web页面的特殊端口会直接报错，原因是端口禁用，请查看

Safari 阻止了一些非标准端口上的内容

https://support.apple.com/zh-cn/HT203772

网站管理员可能注意到 Safari 会阻止一些非标准端口上的 HTTP、HTTPS 和其他内容的请求。有关无效端口号（例如，端口 0 或为 49152 或以上的端口号）的请求也会被阻止。当出现这种情况时，您可能会看到“不允许使用受限制的网络端口”的信息。

本文章已经归档，因此 Apple 将不再对其进行更新。

Web 服务应在标准端口（80、443）或无限制、高范围的端口上提供以获得最佳效果。这些被 Safari 阻止的端口会被保留，以供 Web 通信以外的其他服务（如电子邮件）使用。如此处所述，阻止 Web 浏览器访问这些端口是防止对这些服务恶意访问的一种安全措施。这种功能出现在 Safari 3 或更高版本中。大部分最新版本的 Web 浏览器至少都阻止了部分相同的端口。
